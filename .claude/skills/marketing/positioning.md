# HAL-O Product Positioning

## Three Audiences — Three Hooks

### 1. Hardcore Devs (the "I run 5 agents" crowd)
**Pain**: Claude Code is a black box terminal. One agent, one session, no visibility. Can't see what sub-agents are doing. Can't intervene. Cross-agent communication is blocked by Anthropic. Token costs spiral.

**HAL-O solves**:
- **Own the terminals**: Every agent runs in a visible, interactive xterm.js tab. See output live. Type into any session. Kill, restart, or redirect any agent.
- **Token warfare**: UserPromptSubmit hooks = zero-token keyword shortcuts. `/hal` command center = 20+ actions in 2 keystrokes. Haiku routing for simple agents. Staggered stats, back-face frame skipping, shared geometry. Every optimization documented.
- **Multi-agent orchestration**: HAL-O IS the terminal. Spawn sessions for different projects in tabs. Telegram dispatcher routes messages to the right one. One bot, many brains.
- **Integrated QA + Perf pipeline**: Every visual change auto-spawns QA agent. Every renderer commit triggers perf matrix. CI ownership = never leave dashboard red.
- **Workaround for Anthropic's limitations**: Can't have agents talk to each other? Route through HAL-O's PTY. Can't see agent logs? They're in your terminal tabs. Can't control sub-agents? You own the process.

### 2. Casual Devs (the "I just want it to work" crowd)
**Pain**: Project management is scattered. IDE here, terminal there, git status somewhere else. Claude Code is powerful but intimidating.

**HAL-O solves**:
- **3D holographic mission control**: Your projects orbit a glowing sphere. Health colors, activity bars, commit stats — all at a glance. Not a file tree. A command center.
- **Zero-config import**: Scan → summarize → enlist. Three clicks. It reads your existing Claude configs.
- **Voice control**: CTRL+SPACE push-to-talk. 20 voice profiles. Talk to your projects. Local Chatterbox TTS = free, no API costs.
- **One-click everything**: GPU auto-detection, preset wizard, Settings with icons, `/hal` menu for anything.

### 3. Newcomers (the "what even is Claude Code?" crowd)
**Pain**: Don't know where to start. Terminal is scary. AI coding tools are confusing.

**HAL-O solves**:
- **START_HERE.bat**: One double-click installs everything.
- **Setup wizard**: Detects git, Python, Claude CLI, ffmpeg. One-click install for missing deps. 10s auto-launch countdown.
- **Visual everything**: 3D scene makes it feel like a game, not a tool. Intro camera fly-in. Particle effects. It's exciting.
- **Guided experience**: (Coming) Intro tutorial, demo mode, witty loading messages.

## Killer Feature: Zero-Effort HTML Dashboards

`/hal html perfs` → agent reads your perf data, generates a production-grade interactive HTML dashboard with charts, animations, HAL-O branding, and opens it in your browser. No design skills needed. No prompting a UI. Just describe what you want in 3 words.

The `/html` keyword leverages a frontend UI/UX expert skill (frontend-design plugin) that generates distinctive, non-generic pages. Dark theme, animated charts, hover tooltips, responsive. Every page looks like it was hand-crafted by a designer.

**Frame it honestly**: this costs tokens (the agent reads data + generates HTML). But the ROI is massive — a human designer would spend hours on what this produces in 30 seconds. And the pages are genuinely useful: perf comparisons, Kanban boards, font browsers, rules organizers, marketing pages — all generated on demand from live project data.

**Examples already built this session**:
- Perf comparison with animated bar charts (Mar 23 vs Mar 25)
- Kanban board with drag-and-drop, filters, live clock
- ASCII font browser (570 fonts, search, categories)
- Rules organizer — interactive HTML to inspect, drag-and-drop, and optimize Claude's memory (what goes in CLAUDE.md vs auto-memory, live token estimate)
- Marketing landing page with particle canvas, scroll reveals, comparison tables

## Anti-Patterns (what NOT to say)
- Don't position as "yet another IDE" — it's not an IDE, it's a mission control layer
- Don't compare to VS Code extensions — different category entirely
- Don't undersell the terminal — it's not just pretty 3D, the embedded terminal IS the product
- Don't hide the token cost angle — developers care about money, lead with savings
- Don't make it sound complex — "one double-click to install" is the first impression
