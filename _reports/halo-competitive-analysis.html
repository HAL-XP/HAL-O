Critical Warnings — Anti-Patterns to Avoid

Lessons learned from competitors' missteps. These are not theoretical — they cause real user pain at scale.

⚠️ No Nested Categories (Discord)

Discord's flat channel list is its biggest UX complaint. At 50+ channels, users can't find anything. Fix: Implement category nesting (Dispatcher → Project → Agent) from day 1. Flat scales to ~10 items max.

⚠️ Unnavigable at Scale (Notion)

At 100+ pages, Notion becomes a dumping ground. No hierarchy, no breadcrumbs, search is the only way out. Fix: Enforce 3-level max hierarchy like Terraform Cloud. Add sidebar pin/collapse. Breadcrumbs everywhere.

⚠️ No Virtualization (Slack)

Slack renders all 200+ channels even if only 5 visible. At 500 channels, the DOM becomes unusable. Fix: Use react-window or react-arborist for virtual scrolling at 1000+ items.

⚠️ Confusing Terminology (K8s)

"Namespace" vs "Cluster" vs "Node" — each level uses domain-specific jargon. Fix: Use plain language: "Project", "Workspace", "Agent". Add tooltips for technical terms.

⚠️ Feature Bloat (ClickUp, Jira)

Too many columns, views, filters paralyzes UX. Users don't know what to click first. Fix: Ship 80/20 features. Make advanced stuff opt-in in Settings. Simplify default view.

⚠️ No Keyboard Nav (Teams, Rancher)

Power users can't navigate without mouse. Arrow keys + Enter should drill down and back. Fix: Implement full keyboard nav from day 1 (↓/↑ = move, → = expand, ← = collapse, Enter = select).

⚠️ No Drag-Drop Reordering

Users want to organize their own hierarchy. Missing drag-drop == users feel trapped by the structure. Fix: Implement react-beautiful-dnd or dnd-kit. Test with 100+ items.

⚠️ Slow Tree Rendering (Notion, Rancher)

Re-rendering entire tree on every expand/collapse. At 100+ nodes, frame rate drops to 10 FPS. Fix: Memoize nodes. Use lazy-load + virtualization. Profile with React DevTools.

Key Principles

1. Three-Level Hierarchy (Max)

Dispatcher → Project → Agent → Task. Never deeper. Breadcrumbs at top of detail pane.

2. Master-Detail Responsive

On mobile (<768px): sidebar slides away, detail pane full-width. On desktop: side-by-side. No modal dialogs for context switching.

3. Keyboard-First Navigation

↓↑ = move, → = expand, ← = collapse, Enter = select, ⌘K = search, Esc = back, / = quick-command.

4. Drag-Drop Everywhere

Users drag agents between projects, reorder projects, pin favorites. Drag hint on hover. Undo support.

5. Lazy-Load + Virtualize

Tree only renders visible 30 items at a time. Expand node fetches children. At 10,000 agents, no lag.

6. Real-Time State Sync

Agent running? Icon pulses cyan. Cost updated? Badge animates. No page refresh needed. WebSocket + React context.

Scale Considerations

What works at 5 items breaks at 500. Here's the roadmap.

Scenario Recommended Pattern Library / Tech Notes
5 dispatchers, 10 projects Simple nested list React context No virtualization needed yet. Breadcrumbs + search.
50 projects, 100 agents Lazy-load tree + search react-arborist + fuse.js Expand loads children on demand. Search must be instant (<100ms).
200 agents, 500 tasks Virtual scroll tree + pagination react-window + react-arborist Only render 30 visible nodes. Pagination for tasks view.
1000+ agents, multi-dispatcher Hierarchical search + favorites Algolia / Typesense backend Can't render full tree. Users filter by tag/status. Pin top 5 dispatchers.
10,000+ items (enterprise) Database-backed search + caching PostgreSQL FTS + Redis cache Full client-side tree is impossible. Use server-side filtering.

Implementation Libraries

Best-in-class tools for each component. Tested against 30+ competitors.

react-arborist

Tree UI + Virtualization

Drag-drop tree with 10,000+ items. Built for forest-like hierarchies. Powers VS Code-style trees.

  • Keyboard nav (↓↑→← + Enter)
  • Drag-drop reordering
  • Virtual scroll (only renders visible)
  • Custom node rendering
  • Inline edit names
npm install react-arborist

@radix-ui/context-menu

Right-Click Context Menu

Accessible, keyboard-navigable context menus. Powers professional apps (Figma, Linear).

  • Keyboard nav in menu
  • Submenus (nested actions)
  • Checkboxes + radio groups
  • Custom styling (zero CSS)
  • Trap focus on menu
npm install @radix-ui/react-context-menu

cmdk

Command Palette (⌘K)

Fast, accessible command palette. Used by Linear, Vercel, Stripe. Fuzzy search ~0ms.

  • Instant fuzzy search
  • Nested command groups
  • Keyboard nav only
  • Custom shortcuts
  • Dark mode ready
npm install cmdk

dnd-kit

Drag-Drop Primitive

Modern drag-drop. Better than react-beautiful-dnd. Sortable lists, tree reordering, multi-level.

  • Multi-level tree drag
  • Auto-scroll on drag
  • Touch support
  • Drop animations
  • Undo support
npm install @dnd-kit/core @dnd-kit/sortable

fuse.js

Fuzzy Search

Client-side fuzzy search. Fast at 10,000+ items. Used by VS Code, Slack, Linear.

  • Fuzzy + exact matching
  • Weighted field search
  • Threshold tuning
  • Fast (~50ms at 5000 items)
  • Tiny bundle (5KB gzip)
npm install fuse.js

react-window

Virtual Scrolling

Render only visible items. Powers Discord, Figma, Facebook. Required for 1000+ items.

  • Fixed-size lists
  • Variable-size items
  • Grid layouts
  • Auto-scroll to index
  • Tiny overhead (<2KB)
npm install react-window
