Phase 1: Messaging App Foundation

Vision

Transform from debug tool to a real messaging app. Each agent gets its own chat thread. Left sidebar shows agent list with avatars, last message previews, and unread counts—exactly like Telegram or WhatsApp.

Key Changes

• Remove "ALL" tab—single unified interface

• Left sidebar: agent list with avatars, last message preview, unread badges

• Main area: clean chat thread with message bubbles (left=agent, right=user)

• Agent avatars: HAL=cyan eye, Bob=green briefcase, Karen=pink heart

• Timestamps on every message

• "Typing..." indicator when agent is thinking

Phase 2: Voice & Audio

Vision

Add voice messaging and TTS playback. Users can record messages with waveform visualization. Agent responses auto-play as audio. Voice indicator shows on agent avatar when speaking.

Features

• Tap mic to record: shows waveform in real-time

• Hold-to-record (or tap, release to send)

• Voice message bubbles with playable waveform + play button

• Auto-play agent TTS responses (user can disable per-thread)

• Speaker icon on agent avatar when active

• Visual feedback: waveform animation during playback

Phase 3: 3D Sphere Header (Agent Visualization)

Vision

Elevate the app with the HAL-O sphere at the top. Sphere reacts to voice (pulses on speech). Each agent has a color theme (HAL=cyan, Bob=green, Karen=pink). Sphere animates when switched.

Implementation

• Reuse PbrHoloScene.tsx sphere (or simplified version)

• Sphere color changes per active agent

• Sonar pulse rings on voice activity

• Status text: "Online", "Typing...", "Listening..."

• Smooth transitions between agent themes

Phase 4: Rich Media & File Sharing

Vision

Users can share images, documents, and files. Agents can send HTML pages and rich previews. Images render inline in chat. Documents show as linked cards.

Features

• File upload: tap + button, select from gallery/device

• Image preview: inline thumbnail with fullscreen modal

• Document card: name, size, download link

• HTML render: agent can send mini web pages (isolated iframe)

• Clipboard integration: paste images directly into chat

• Drag & drop (desktop PWA)

Phase 5: Settings & Personalization

Vision

Users can customize agents, theme, and notification preferences. Clean Apple-style settings UI. Support agent creation and editing (name, voice, personality, avatar, color).

Features

• Agent editor: name, avatar, color, voice selection, personality slider

• Theme selector: light/dark, accent color picker

• Notification toggles: sound, vibration, desktop notifications

• Per-agent settings: auto-play TTS, notifications on/off

• Export/import chats (JSON backup)

• Server connection settings (IP, port, token)

Technical Architecture

Frontend Stack
  • Framework: React 19 + TypeScript
  • PWA: vite-plugin-pwa
  • Voice: transcribe.py + tts.py
  • Audio API: Web Audio API, Recorder.js
  • 3D: Three.js (sphere from HAL-O)
  • Styling: CSS-in-JS (styled-components or Tailwind)
Backend Communication
  • Server: PC running HAL-O (Electron)
  • Protocol: WebSocket (real-time chat)
  • Auth: Token-based (localStorage)
  • Storage: IndexedDB (offline cache)
  • File Upload: FormData POST
  • Voice Stream: WebRTC or chunked upload
