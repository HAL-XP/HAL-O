HAL-O Tree Architecture

Project Management & Dispatcher Hierarchy Redesign

⬜ Current Architecture (Before)

The current HAL-O system uses a flat project discovery model with limited hierarchy management. Projects are auto-scanned from the filesystem, grouped only via localStorage, and all managed through a single monolithic dispatcher.

Current Structure

📁 Auto-Discovery
Projects scanned from ~/.claude/projects/ on app load
📋 Flat Groups
Groups stored in localStorage only, no nesting or persistence
🤖 Single Dispatcher
Monolithic dispatcher manages all projects and agents
🚫 Hidden Projects
Exclusion via .claude_ignore, not elegant or discoverable
💬 Single Bot
One Telegram bot per HAL-O instance, no sub-dispatcher routing
❌ No Hierarchy
All projects treated equally, no parent-child relationships

Limitations

⚠️ Auto-scanning creates noise (dozens of personal projects visible at launch) • Projects can't be organized by context (work vs personal) • Sub-agents can't have independent Telegram bots or avatars • No lazy loading — all projects loaded upfront • Hidden projects feel like a hack rather than first-class citizens

🌳 Proposed Tree Architecture (After)

Restructure HAL-O around an explicit hierarchy: dispatchers as first-class citizens that own projects and groups, with each dispatcher capable of independent Telegram integration, voice profiles, and agent-to-agent communication.

Node Types & Styling

⚡ Dispatchers (Root + Sub)
Can own projects, spawn agents, manage Telegram bots, set voice profiles. Each has independent memory, settings, and command routing. Shown in cyan with glow effect.
📦 Projects
Leaf or parent nodes with code. May be children of dispatchers or groups. Stack badge shows primary language. Shown in green.
📁 Groups
Organizational folders (never contain code). Can nest other groups or projects. Useful for large dispatchers. Shown in purple italic.
✨ Expansion
Click any node to expand/collapse children. State persisted in localStorage. Smooth animations. Hover highlights with instant feedback.

💡 Key Design Decisions

🤖 Dispatchers ARE Projects
Each dispatcher (e.g., Karen, Bob) is itself a project with a CLAUDE.md, memory, git history, and the ability to run code and spin up agents. Breaks the false distinction between "manager" and "worker."
💬 Each Dispatcher Gets a Bot
Every dispatcher (including root HAL-O) has its own Telegram bot token. Messages route to the appropriate dispatcher's terminal. Root can access all children; children can escalate to root. Sub-dispatchers show as separate avatars in Halo Chat.
🎯 Projects Are Opt-In
No more auto-scanning. Users explicitly import projects via the "Manage Projects" page. Reduces noise, improves control. Hidden projects are just... not imported. Simple and discoverable.
🌐 Root Can See Everything
HAL-O root dispatcher has global visibility into all child projects and dispatchers. Can query stats, run cross-project commands, act as orchestrator. Children cannot see siblings unless explicitly connected.
⚡ Lazy Loading
Projects loaded on-demand when first accessed. Shows loading state, then activates terminal and Git data. Startup time remains constant regardless of tree size.
🎨 Voice & Language per Dispatcher
Each dispatcher can have different voice profile (soft, butler, dramatic) and language. Karen might speak French & soft, Bob might speak English & formal. Settings inherited by children unless overridden.
🔐 Permission Model
Root reads all projects. Dispatchers own their branches. Users can grant cross-dispatcher access (e.g., Bob can read Karen's blog). Enforced via RBAC in memory files.
💾 Persistence
Tree structure, expansion state, and settings stored in a machine-readable JSON file (`.claude/hal-o-tree.json`). Synced to git so the tree is version-controlled. Onboarding flow auto-initializes this.

Import Flow

Click "+ Import Project" in the left sidebar to trigger a guided workflow:

1. Folder Picker — Navigate to a local folder
2. Auto-Detect — Scan for package.json, Cargo.toml, go.mod, etc. Suggest primary language badge
3. Name & Assign — User confirms project name and chooses which dispatcher/group to assign it to
4. Confirm — Show summary, offer to pin favorite, then add to tree
5. Done — Project appears in tree, terminal can be opened

💬 Halo Chat Integration

The tree structure maps naturally into Halo Chat's sidebar organization. Each dispatcher becomes an avatar that can be interacted with independently.

Sidebar Mapping

👤
Dispatcher = Avatar
Each dispatcher (Karen, Bob, HAL-O Root) appears as a selectable avatar in Halo Chat sidebar. Click to switch which dispatcher you're chatting with.
📁
Groups = Sections
Folder groups expand in sidebar to show their projects as child items. Can collapse entire sections to reduce clutter.
📦
Projects = Commands
Projects under a dispatcher become accessible via `@project-name` prefix in chat. Routes voice/text commands to that project's terminal.
🔗
Agent-to-Agent Chat
Dispatchers can message each other. Karen can ask Bob for stats, Bob can delegate work to Karen. Full inter-process communication via Telegram or internal IPC.

Message Routing Logic

// Example: User sends voice command to Karen User: "Karen, deploy the blog-cms" ↓ routes to Karen's terminal ↓ Karen parses "@blog-cms deploy" ↓ activates blog-cms project terminal ↓ executes: npm run deploy ↓ returns stdout to Halo Chat // Example: Root queries all dispatchers User: "HAL-O, get status from all teams" ↓ HAL-O (root) opens IPC channels to Karen + Bob ↓ sends "GET /status" to each ↓ aggregates results ↓ displays unified dashboard

📈 Implementation Phases

Phase 1
Opt-In Project Management Active
  • Replace auto-scan default with "Manage Projects" page
  • Implement folder picker + language auto-detection
  • Store projects in `.claude/hal-o-projects.json` (new file)
  • Update ProjectHub to read from JSON instead of filesystem scan
  • Remove .claude_ignore logic (no longer needed)
  • Backfill existing users: auto-import from ~/.claude/projects/ on first launch
Phase 2
Tree Structure with Groups & Nesting Planned
  • Create `.claude/hal-o-tree.json` schema with dispatcher/group/project nodes
  • Build interactive tree UI (expand/collapse, drag-to-reorder)
  • Implement tree-based project loading in ProjectHub
  • Add group management: create, rename, delete, move projects
  • Persist tree state in localStorage (expanded nodes)
  • Git-check hal-o-tree.json so tree is version-controlled
Phase 3
Sub-Dispatchers with Independent Bots Planned
  • Create dispatcher project archetype (CLAUDE.md template, memory dir, git repo)
  • Implement sub-dispatcher terminal activation via IPC
  • Each dispatcher gets own Telegram bot token in tree.json
  • Route incoming Telegram messages to correct dispatcher's terminal
  • Add Halo Chat avatar switching (Karen, Bob, HAL-O)
  • Sub-dispatcher memory separate from root
Phase 4
Agent-to-Agent Communication Planned
  • Add IPC channels for dispatcher-to-dispatcher messaging
  • Implement @prefix routing in chat (e.g., Karen → @bob get-stats)
  • Create message queue system for offline/async messages
  • Add permission layer (who can talk to whom)
  • Telegram relay: messages can flow through multiple dispatchers
  • Build Halo Chat inter-agent display (conversation threads)
Phase 5
Lazy Loading & Auto-Scaling Planned
  • Load projects on-demand (not at app startup)
  • Show loading spinner when opening a project terminal
  • Cache git data (stats, branches) per project
  • Auto-scale tree rendering for 50+ projects
  • Implement search/filter to find projects by name/tag
  • Add "favorites" star system for quick access

🔌 Data Structures & IPC Channels

`.claude/hal-o-tree.json` Schema

{ "version": "1.0", "rootDispatcherId": "hal-o", "dispatchers": { "hal-o": { "name": "HAL-O Root", "type": "dispatcher", "telegramBotToken": "env:TELEGRAM_MAIN_BOT_TOKEN", "voiceProfile": "auto", "language": "en", "memory": ".claude/hal-o-memory", "children": ["karen", "bob", "my-games", "hal-o"] }, "karen": { "name": "Karen", "type": "dispatcher", "projectPath": "~/code/karen", "telegramBotToken": "env:TELEGRAM_KAREN_BOT_TOKEN", "voiceProfile": "soft", "language": "fr", "memory": "~/code/karen/.claude/memory", "children": ["personal-website", "blog-cms"] } }, "projects": { "client-api": { "name": "client-api", "type": "project", "path": "~/code/client-api", "language": "node", "starred": true } }, "groups": { "my-games": { "name": "My Games", "type": "group", "children": ["ue-phone-game", "offline-game"] } } }

New IPC Channels

tree:load
Load entire tree from hal-o-tree.json. Returns parsed tree structure.
tree:save
Write tree to hal-o-tree.json. Validates schema before writing.
project:import
Scan folder, auto-detect language, return metadata for user to confirm.
project:activate
Open project terminal, load git data, show in hub. Lazy-load if needed.
dispatcher:list-children
Get all projects and groups under a dispatcher. Recursive or flat.
dispatcher:send-message
Send text/voice message from one dispatcher to another. IPC queue if offline.
